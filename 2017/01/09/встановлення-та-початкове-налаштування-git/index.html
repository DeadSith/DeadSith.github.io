Встановлення та початкове налаштування git

2017-01-09 · 408 words · 2 minute read
git
git · starting · Українською

Вступ

Git — розподілена система керування версіями файлів та спільної роботи. Основними перевагами цієї системи є простота в користуванні та широкий функціонал.
Це керівництво познайомить вас з основами роботи з Git та сервісом GitHub. Ця частина описує встановлення та використання Git з Windows. Користувач Linux можуть пропустити цей крок, оскільки в більшості дистрибутитів Git встановлено за промовчанням.

Встановлення

Завантажити Git для Windows можна за наступним посиланням: https://git-scm.com/download/win. Завантаження має початися автоматично; якщо цього не відбулося необхідно вибрати один з варіантів Git for Windows Setup. Після завантаження запускаємо установочний файл і натискаємо Next > поки не з’явиться У цьому і всіх наступних меню необхідно вибирати сам такі елементи, як на скріншотах.

Перший запуск

Для подальшої роботи необхідно створити навчальний проект. Я, наприклад, створив папку GitTutorial. Тествовий проект доступний в моєму профілі на GitHub. Ви можете створити папку з будь-якою назвою, але я рекомендую використовувати зрозумілі назви, а не випадкові послідовності символів. Після створення папки, відкриємо її. Далі відкриємо контекстне меню. Якщо Git встановлений коректно, то в ньому мають з’явитися нові пункти: Git Bash Here та Git Gui Here. Вибираємо Git Gui Here. Відкриється наступне вікно:
Якщо шлях до папки або ім’я користувача містить неанглійські символи, то вони будуть відображатися некоректно.
Виправимо цю помилку: 1. Натискаємо правою кнопкою миші на ярлик у лівому верхньому куті вікна.
2. У меню, що відкрилося, вибираємо пункт Options…
3. Вибираємо підменю Text.
4. У списку Locale вибираємо uk_UA(або ru_RU на російській версії Windows).
* Character set, за необхідності, змінюємо на UTF-8.
5. Натискаємо Apply.
6. Натискаємо Save та перезавантажуємо Git Bash.

Початкове налаштування

На даному етапі у вас уже має бути акаунт на GitHub. Якщо його немає, то необхідно зареєструватися. У більшості випадків достатньо безкоштовного акаунту. На момент написання студенти можуть отримати безкоштовний повний акаунт на Github як частину Student Developer Pack.
Далі необхідно налаштувати Git. Тут і надалі, рядки консольного коду, які починаються з $ вводить користувач. Рядки, які не мають цього символу на початку - це результат роботи Git, їх вводити не треба. Сам символ $ користувач вводити не має, це автоматично робить Bash. Git Bash підтримує копіювання і вставлення тексту. Для вставлення необхідно натиснути Shift+Ins, копіювання - Ctrl+Ins. Для налаштування необхідно ввести наступні команди


$ git config --global user.name "<Ваше ім`я>"
$ git config --global user.email <e-mail, який ви використали при реєстрації на GitHub>

E-mail має збігатися для коректної роботи GitHub.

Завершення

На цьому установка і перше налаштування Git завершене. У наступній частині я познайомлю вас з основами роботи з Git.
